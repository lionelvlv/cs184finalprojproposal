Splat Painter

Patrick Lee, Lionel Verano, Jordan Argel, Eric Lin

Link to webpage: https://lionelvlv.github.io/cs184finalprojproposal/

Summary

The purpose of Splat Painter is to modify the hue and saturation of gaussians within a gaussian splatting scene. This will allow a user to use their mouse cursor as a brush to "paint" over the existing scene similar to brush tools in raster graphics editors. We’ll be using Nerfstudio’s gsplat and Splatfacto as a starting point and build from these tools.

Problem Description

The problem posed is most gaussian splatting viewers have limited ways to edit the scene currently being viewed. We want to allow the user to change the colors of objects within the scene whilst being able to preserve the lighting already present within the scene. This opens up the possibilities to imagine real life environments with different colors applied. For example, one could change the colors of the walls of a room or change the fabric color of a couch. Or even see real life through an impossible manner such as making grass look like a rainbow. One may even be able to reconstruct an empty canvas and paint the canvas through our program.

A few challenges in this project would be learning to understand the gaussian splat scenes are structured and gsplat and splatfacto's code base. We expect to read through a lot of gsplat's documentation for this project in order to solve it. We then need to figure out a method to both capture the area of splats in a bounded volume for our cursor to act as a brush and then apply color shifts to the gaussian's color data to change its appearances.

Goals and Deliverables

What we plan to deliver

To show results, we'll show images of unmodified splat scenes and resulting images with different colors applied to the scene

What we hope to deliver

  • We'll also show a screen recording demo of us using the tool to paint in real time within Splatfacto
    • This will be a modified version of Splatfacto which includes our painting tool
  • Ideally, our painting tool should be able to paint in real time. Performance may vary wildly depending on the GPU used due to the nature of the gaussian splatting hardware requirements
    • Our project shouldn't noticeably slow down the existing nerfstudio viewer's performance
